Help Topics

How to Use

With your device physically linked to the preferred COM port. Connect the emulator using the Connect Serial button in the CONTROLS tab on the right.

It's key to ensure the COM port, connections and serial settings match!

When connected the status indicator will change from OFFLINE to IDLE, the virtual printer is now ready to receive serial print commands.

Any serial RS232 device can be connected, including Organiser I' & II's and many other devices!

Printing

Using

The emulator accepts plain ASCII text and classic control codes from any device on the connected serial port. See the Printer Control Codes section for a full list.

Printing Text

When using an Organiser II over a COMMS / RS232 Link, standard OPL LPRINT commands will output directly to the scrolling thermal print roll.

Printing Graphics

To design custom graphics for the Organiser II, use the companion-app: Printer Graphics RLE Compressor.

To print your graphics from an Organiser II, use the custom OPL GPRINT: stub to replace the classic printer's GPRINT command (See CONTROLS to transfer it to an Organiser II).

Hardware & Conflict Warnings

A serial COM port can only be connected to one application on the host machine. Such as a Serial-to-USB convertor or internal serial COM port.

Connect your serial device e.g. the Organiser's serial COMMS Link to the selected serial COM port or Serial-to-USB convertor on the host computer.

The browsers Web Serial will try to set the default baud rate settings. You may need to set your host computers serial port settings to match ( e.g. 9600 Baud, 8 Data Bits, No Parity, and 1 Stop Bit ).

Important: Ensure no other software (terminal programs, ORG-LINK or OPK transfers) is holding the serial port open, as standard OS ports can only be accessed by one app / program at a time.

Printer Control Codes

The emulator supports hardware command formatting:
• CHR$(9): Tab (move to centre of page)
• CHR$(10): New Line (\n)
• CHR$(12): Form Feed (3 line)
• CHR$(13): Carriage Return (\r)

• CHR$(14): Double Width (20 columns)
• CHR$(15): Normal (40 columns)
• CHR$(23): Narrow (60 columns)
• CHR$(22): Very Narrow (80 columns)
• CHR$(21)/(24): Underline On/Off
• CHR$(17)/(18): Double Height On/Off (if flow control filter is disabled)

Keyboard Shortcuts

• Alt + G: Toggle Guide Panel (Left)
• Alt + C: Toggle Controls Panel
• Alt + M: Toggle Print Modes Panel
• Alt + O: Toggle Operations Panel
• Alt + H: Toggle Help Panel
• Alt + S: Toggle Serial Connection
• Alt + T: Print Test Page
• Alt + F: Feed Paper roll
• Delete: Clear Paper Roll

PC Connections

Devices other than an Organiser II may need custom wiring. A classic Organiser II COMMS/RS232 link is designed for direct connection to a 25 way D-Type COM port connection.

If using custom connections you will need to connect at least the following PC COM port pins:
25 Way - pins 2,3 & 7
or
9 Way - pins 2,3 & 5

GPRINT OPL Routine Transfer

Note: The GPRINT: stub is only required to support graphics printing. Text printing works out-of-the-box. Refer to the GUIDE tab for more information.
Active Emulation Link Paused: An active printer emulation session is running. Proceeding will suspend the printer emulation logically so we can use the link for files.

1. Set Up Organiser II COMMS settings

Ensure your Organiser is configured to use default settings: 9600 Baud, No Parity, 8 Data bits, 1 Stop bit, XON/XOFF flow control (Note: only 9600 baud and default settings are currently supported by the emulator).

2. Prepare Organiser II for Receive Mode

  1. Press ON/CLEAR to go to the main menu.
  2. Find and select the COMMS menu option, then press EXE.
  3. Select RECEIVE from the submenu and press EXE.
  4. Select PROCEDURE and press EXE.
  5. When the screen shows RECV A:, type the name GPRINT and press EXE.
The Organiser screen will show a blank screen waiting for bytes. When you click Start Transfer, select the correct COMMS link serial port from the browser's Serial Port Access Request popup, and the OPL source code will begin transferring automatically.

3. Transfer Progress

Transfer Completed Successfully!

Follow these steps on your Organiser to translate/compile the routine:

  1. Press ON/CLEAR 3 times to return to the Main menu.
  2. Select PROG from the main menu and press EXE.
  3. Select EDIT, type GPRINT, and press EXE.
  4. Inside the OPL code editor, press MODE and select TRAN to translate the stub.
  5. Once translated, select SAVE to store the compiled procedure.
You are good to go! The GPRINT: graphics helper is now compiled and ready on your Organiser II.
